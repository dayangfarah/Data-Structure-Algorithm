# SECJ2013: Data Structures and Algorithms

## 📌 Course Overview
* **Faculty:** Software Engineering/Computing
* **Course Code:** SECJ2013
* **Academic Session/Semester:** 2024-2025/1
* **Credit Hours:** 3
* **Pre/Co-requisite:** Programming Technique 1, Programming Technique 2

---

## 📖 Course Synopsis
This course emphasis on data structure concepts theoretically and practically with detail algorithms for each of data structure. Students will learn abstract data type concepts using class and apply the concept in the implementation of data structures. Apart from it, student will learn recursive as a programming style and algorithm efficiency analysis with Big O notation. Various sorting and searching techniques will be discussed as data structure operations. Efficiency analysis of each algorithm will also be explained. Further, students will be exposed to linear data structures such as linked lists, stack and queue, while tree and binary search tree will be introduced as non-linear data structures. Along the course, students should be able to implement and apply the theory and concepts of data structure in the assignments and mini project which are conducted in group.

## 💭 Reflection

### What I Learned
Taking SECJ2013 transformed the way I approach code execution and memory management. I moved past simply writing functional software to understanding exactly how data is structured, cached, and manipulated behind the scenes. Mastering abstract data types through C++ classes, analyzing algorithmic efficiency with Big O notation, and tracing complex recursive algorithms provided a rigorous baseline for identifying suboptimal operations. Gaining deep visibility into linear architectures (linked lists, stacks, queues) and non-linear designs (trees and binary search trees) completely changed my perspective on computational efficiency.

### Challenges & Growth
The learning curve rose sharply when transitioning from standard array-based storage to dynamic pointer manipulations within manual linked list configurations. Debugging runtime fragmentation during structured data operations, node deletions, or parsing expressions using manual stack stacks was heavily testing at first. Furthermore, shifting from baseline sorting models like Bubble or Insertion sort to advanced divide and conquer patterns like Quick and Merge sort required structural adaptation, strengthening my algorithmic layout capabilities. 

### Future Application
The paradigms developed over the duration of this module are directly vital to my trajectory in system design and backend engineering. Writing memory-safe architectures and organizing massive database systems depends entirely on selecting optimal data setups. Moving forward, I will continuously apply Big O profiling to avoid processing bottlenecks, leverage specific search hierarchies like Binary Search Trees to optimize access layers, and format clean code routines that support stable execution.
